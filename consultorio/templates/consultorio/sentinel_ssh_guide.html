Guía de Conexión Remote SSH

Configura Cursor para reparar errores en vivo directamente en el servidor de producción

FLUJO DE AUTOCURACIÓN
Error detectado Gemini analiza Sentinel notifica SSH + Cursor Reparado
1
Instalar la extensión Remote SSH en Cursor

Cursor usa el mismo sistema de extensiones que VS Code. Instala "Remote - SSH" de Microsoft.

# En Cursor, presiona Ctrl+Shift+X (Extensiones)
# Busca: "Remote - SSH"
# Instala la extensión de Microsoft

# También instala:
# "Remote - SSH: Editing Configuration Files"
2
Configurar Llave SSH de la VPS

Genera una llave SSH y agrégala a tu servidor remoto.

# Generar llave SSH (solo una vez)
ssh-keygen -t ed25519 -C "tu-email@gmail.com"

# Copiar la llave pública
type %USERPROFILE%\.ssh\id_ed25519.pub

# Agregar la llave al servidor remoto:
# 1. Accede al panel de tu VPS o al archivo authorized_keys
# 2. Pega el contenido de id_ed25519.pub
# Configurar archivo SSH (~/.ssh/config)
Host prislab-server
  HostName XX.XX.XX.XX
  User jonathan
  IdentityFile ~/.ssh/id_ed25519
  StrictHostKeyChecking no
3
Conectar Cursor al Servidor

Una vez configurado, conéctate al servidor desde Cursor.

# En Cursor:
# 1. Presiona Ctrl+Shift+P
# 2. Escribe: "Remote-SSH: Connect to Host"
# 3. Selecciona "prislab-server"
# 4. Se abrirá una nueva ventana conectada al servidor

# 5. Abre la carpeta del proyecto:
# Archivo → Abrir Carpeta → /app/

# Si usas Docker, entra al contenedor:
docker exec -it prislab_app bash
4
Reparar con PRIS Sentinel + Cursor

Cuando Sentinel detecte un error, así es como lo reparas:

  1. Ve al Panel Sentinel y busca la incidencia con estado Pendiente
  2. Haz clic en
  3. Se abrirá un modal con el prompt completo que incluye:
    • Archivo exacto y línea del error
    • Código que falla vs código propuesto
    • Instrucciones SSH
  4. Copia el prompt → Pégalo en el chat de Cursor (conectado via SSH)
  5. Cursor abrirá el archivo, irá a la línea y aplicará la corrección
  6. Gunicorn se reinicia automáticamente → Error reparado en vivo
Resultado: El error se corrige directamente en producción sin necesidad de hacer git push, rebuild de Docker ni redeploy. Los cambios se aplican al instante.
5
Verificar y Cerrar la Incidencia

Después de aplicar la corrección:

# Verificar que no haya errores de importación
python manage.py check --deploy

# Reiniciar Gunicorn sin downtime
kill -HUP 1

# Verificar que la URL afectada ya no da error
curl -s -o /dev/null -w "%{http_code}" https://tu-dominio.com/URL_AFECTADA

Vuelve al Panel Sentinel, actualiza el estado a Solucionado y agrega notas de resolución.
